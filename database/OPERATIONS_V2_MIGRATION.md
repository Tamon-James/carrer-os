# Operations V2 既存データ移行手順

この手順は、現在サーバー上で利用しているMySQLデータを保持したまま、新しいメモ・セッション・アーカイブ機能へ移行するためのものです。

## 移行される既存データ

- ユーザー、企業、応募案件、予定、タスク、メモ、面接ログ、資料はそのまま保持されます。
- 既存の選考フローには、応募案件から対応企業を自動設定します。
- 既存の面接対策・カンペは、元データを残したまま統合メモへコピーします。
- 既存の面接ログは、引き続きセッションログとして参照できます。

## 1. 必ずバックアップする

phpMyAdminで現在の本番DBを選択し、`エクスポート`からSQL形式で全テーブルを保存してください。

移行中は、可能であれば一時的にアプリの利用を停止してください。

## 2. 新しいコードを配置する

新しい`public`、`src`、`views`、`database`をサーバーへ配置します。

レンタルサーバーの公開ディレクトリが`public_html`の場合、ローカルの`public`配下を`public_html`へ配置します。

`Carrer_site_private/config.php`と`Carrer_site_private/uploads`は上書き・削除しないでください。

## 3. DBマイグレーションを適用する

### PHP CLIを利用できる場合

```bash
php database/migrate.php
php database/verify_operations_v2.php
```

### phpMyAdminを利用する場合

1. phpMyAdminで現在利用中の本番DBを選択する
2. `database/migrations/003_operations_v2.sql`をインポートする
3. `schema_migrations`テーブルへ以下を追加する

```sql
INSERT INTO schema_migrations (version)
VALUES ('003_operations_v2.sql');
```

`003_operations_v2.sql`は一度だけ実行してください。同じDBへ再実行しないでください。

## 4. 移行後に確認する

- 既存ユーザーでログインできる
- 企業・応募案件・予定・タスク件数が移行前と同じ
- 選考フローが企業詳細に表示される
- `メモ・カンペ`画面に既存メモと既存カンペが表示される
- セッションモードで既存カンペを確認できる
- 過去企業一覧を開ける

## ロールバック

問題が発生した場合は、新しいコードの利用を停止し、手順1で取得したSQLバックアップを空のDBへ復元してください。

移行後に作成された新規データがある場合、バックアップ時点より後のデータはロールバックで失われます。
